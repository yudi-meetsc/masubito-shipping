## 棚番による振り分けルール

| 棚番値 | 振り先 |
|---|---|
| 1 | 高知注文データ |
| 2 | 堺メイン or 堺ゆうパケット判定へ |
| 3 | 堺クーラー |
| 4（内部で2に正規化） | 堺ゆうパケット(単独4) or 堺メイン(2+4混在) |

棚番4は categorizedRows[2] に push されるが、row の棚番列は変更しない。
createSakaiOrder 内で row の棚番原値を使って分類。

堺の詳細振り分け（管理番号でグループ化後）:
- グループ内の棚番が全て `2` → **堺メイン**
- グループ内の棚番が全て `4` → **堺ゆうパケット**
- `2` と `4` が混在 → **堺メイン**
- 棚番 `3` → **堺クーラー**（グループ化せず個別行）

**重要**: 堺メイン・ゆうパケットはグループ内の先頭行1行のみを出力する（1管理番号＝1行）。
堺クーラーは個別行をそのまま出力する。

トータルピック表: 棚番 `1` 以外を全て集計（JANコード or 商品コードでユニーク集計、数量合算）

ピッキングリスト: 堺系（棚番2/3/4）を管理番号でグループ化。棚番3は `管理番号_3-N` キーで別ページ。

## セル値の特殊処理

| 列 | 処理 |
|---|---|
| 管理番号 | 8桁ゼロ埋め、文字列型 |
| 配送時間帯コード | 2桁ゼロ埋め、文字列型 |
| JANコード / 商品コード | JANがあればJANを採用、文字列型 |
| 届け先ＴＥＬ | `+81XX...` → `0XX...` に変換、文字列型 |
| 届け先郵便番号（堺系） | 文字列型 |
| 届け先住所２（日付っぽい文字列） | 文字列型 |

## 堺系 住所処理

```
全角マイナス「−」→ 半角「-」（住所・電話・郵便番号すべて）
届け先都道府県 + 届け先住所１ + 届け先住所２ を結合
→ 16文字ごとに お届け先住所１ / お届け先住所２ / お届け先住所３ に分割
届け先氏名 16文字超 → 先頭16文字を お届け先名称１、残りを お届け先名称２ へ
```

## ピッキングリスト PDF レイアウト

A4縦、余白0.25インチ。各注文グループ1ページ（最大100ページでPDF分割）。
reportlab CIDフォント `HeiseiKakuGo-W5` を使用。

グリッド: 34列 × 40行、各セル幅21pt・高さ21pt。

```
B2:  "管理番号：{8桁ゼロ埋め}"    AA2: "日付：{YYYY年MM月DD日}"
B2:AG2   下罫線

B4:AG8   お届け先名ブロック（四辺罫線）
  B4: "お届け先名"
  C6: "{届け先氏名} 様"

B10:AG17 お届け先情報ブロック（四辺罫線）
  B10: "お届け先情報"
  C12: {届け先郵便番号}
  C16: {届け先ＴＥＬ}（string型）

B19: "商品名"  AD19: "数量"  （ヘッダー行）

B20〜: 商品行（各行 = JANorCode + ' ' + 標準商品名 + ' ' + 属性１名 + ' ' + 属性２名）
  偶数行(0始まり) #FFFFFF / 奇数行 #F2F2F2
  各行に四辺罫線

B{totalRow}: "計"  AD{totalRow}: 合計数量
B{totalRow}:AG{totalRow} 四辺罫線
```

**商品行の集計なし**: グループ内の各CSV行が1商品行（行ごとに数量を表示）。
棚番3の各行は独立ページ（管理番号_3-N キー）。
